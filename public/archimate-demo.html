ArchiMate Description Format Implementation ✅ COMPLETE

Issue #429 Requirements

The ArchiMate extraction must follow the Example Description Format:

  • Include the LM's FHIR id
  • Hierarchical list of all fields with cardinality in square brackets
  • Data type in parentheses
  • Use indentation with tree characters (└──, │) for hierarchy

Before vs After Implementation

❌ Before (Old Format)

Logical model derived from PatientData.fsh

✅ After (New Format)

id: PatientData Fields: - name [0..1] (string) - address [0..*] (Address) └── line [0..*] (string) └── city [0..1] (string) └── postalCode [0..1] (string) - managingOrganization [1..1] (Reference(Organization)) - birthDate [0..1] (date) - gender [0..1] (code)

Key Features Implemented

  • ✅ FHIR ID Display: Shows the logical model ID at the top
  • ✅ Hierarchical Structure: Uses tree characters (└──) for nested elements
  • ✅ Cardinality Brackets: Shows [0..1], [1..1], [0..*] format
  • ✅ Data Type Parentheses: Shows (string), (Address), (Reference) format
  • ✅ Monospace Font: Better alignment and readability
  • ✅ Line Break Preservation: Uses white-space: pre-line
  • ✅ Dark/Light Mode: Theme-aware CSS variables
